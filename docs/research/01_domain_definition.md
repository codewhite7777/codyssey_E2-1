# 01. E-Commerce 도메인 정의 및 문제 레벨 정리

> 목적: AI 서비스 아이디어를 발굴하기 전, "이커머스란 무엇이며, 어느 지점에서 어떤 사람이 어떤 문제를 겪는가"를 구조화하여 문제 발굴의 좌표계를 만든다.

---

## 1. E-Commerce 정의

### 1.1 협의의 정의

> **인터넷·모바일·디지털 채널을 매개로 상품 또는 서비스를 거래하는 모든 상행위(commercial transaction)를 의미한다.**

협의의 정의는 _거래(transaction) 그 자체_ 에 초점을 둔다. 즉 상품·서비스에 대한 **주문 생성과 결제 완료**를 핵심 사건(event)으로 본다. 거래를 둘러싼 마케팅·물류·CS는 배제하고, "디지털 채널 위에서 매수–매도가 합의되어 결제로 종결되는 행위"만을 가리킨다.

#### 구성 요소

| 요소 | 설명 |
|------|------|
| **거래 주체 (Buyer / Seller)** | 디지털 신원(계정·사업자 정보)으로 식별되는 매수자·매도자 |
| **거래 객체 (Goods / Service)** | 물리 상품, 디지털 콘텐츠, 구독 서비스, 티켓·예약권 등 |
| **거래 매개 (Channel / Platform)** | 웹사이트·모바일 앱·오픈마켓·메시징·라이브·API 자동거래 등 |
| **결제 수단 (Payment)** | 카드·간편결제·계좌이체·BNPL·암호자산 등 디지털 결제 |
| **거래 기록 (Record)** | 주문번호·영수증·트랜잭션 로그 등 디지털로 보존되는 증빙 |

#### 오프라인 상거래와의 핵심 차이

1. **비대면성** — 매수자·매도자가 물리적으로 동일 공간에 존재하지 않는다.
2. **시공간 비제약성** — 24/7 거래 가능, 국경을 넘는 도달.
3. **데이터 추적성** — 모든 거래가 디지털 로그로 남아 분석·재활용된다.
4. **알고리즘 매개성** — 검색·추천·랭킹·동적가격 등 알고리즘이 거래 성사에 직접 개입한다.

#### 협의에서 _제외_ 되는 영역 (→ 1.2 광의에서 포함)

- 거래 _이전_ 단계: 상품 기획, 마케팅, 콘텐츠 제작, 사용자 탐색·비교·의사결정
- 거래 _이후_ 단계: 풀필먼트, 배송, CS, 반품·교환, 리뷰·재구매

### 1.2 광의의 정의
거래 행위 그 자체뿐만 아니라, 거래를 둘러싼 **전 가치사슬(End-to-End Commerce Value Chain)**을 포함한다.
즉 _상품 기획 → 콘텐츠화 → 노출/검색 → 의사결정 → 결제 → 풀필먼트 → 배송 → CS → 리뷰/재구매_ 의 순환 루프 전체.

### 1.3 거래 주체별 분류

| 유형 | 설명 | 대표 사례 |
|------|------|-----------|
| **B2C** | 기업이 일반 소비자에게 판매 | 쿠팡, 11번가, 아마존 |
| **B2B** | 기업 간 거래 | 알리바바, 이지웰비즈 |
| **C2C** | 개인 간 거래 | 당근, 번개장터, 중고나라 |
| **B2B2C** | 플랫폼이 셀러–소비자를 중개 | 네이버 스마트스토어, 무신사 |
| **D2C** | 브랜드가 소비자에게 직접 판매 | 마켓컬리(자체 PB), 나이키닷컴 |
| **O2O** | 온·오프라인 연계 | 배달의민족, 마켓컬리 새벽배송 |

### 1.4 채널·포맷별 분류

- **종합몰 / 오픈마켓** — 쿠팡, 11번가, G마켓
- **전문몰(버티컬)** — 무신사(패션), 마켓컬리(신선), 오늘의집(인테리어)
- **소셜·콘텐츠 커머스** — 인스타그램 숍, 카카오톡 선물하기
- **라이브 커머스** — 네이버 쇼핑라이브, 쿠팡 라이브
- **메타 / 비교 커머스** — 다나와, 에누리
- **구독·정기배송** — 와이즐리, 베이글코드

### 1.5 카테고리별 특수성

| 카테고리 | 특수 페인포인트 |
|----------|----------------|
| 패션·뷰티 | 사이즈·핏·색감 미스매치, 반품률 높음 |
| 신선식품 | 콜드체인, 유통기한, 재고 회전율 |
| 가전·디지털 | 스펙 비교 복잡성, 가격 변동성 |
| 중고·리커머스 | 신뢰·진위·가격 책정 |
| 럭셔리 | 정품 인증, 글로벌 직구 |
| 디지털 콘텐츠·구독 | 이탈(Churn), 개인화 |

## 2. E-Commerce 가치사슬 레이어

문제는 가치사슬의 어느 **레이어**에서 발생한다. 어느 레이어에 개입할지가 곧 서비스의 정체성이다.

```
[L1 소싱·기획] → [L2 등록·콘텐츠] → [L3 검색·노출·랭킹] → [L4 광고·마케팅]
       ↓                                                          ↓
[L11 리뷰·재구매] ← [L10 CS·반품] ← [L9 배송·라스트마일] ← [L8 재고·풀필먼트]
       ↑                                                          ↑
       └──────── [L7 결제·금융] ← [L6 의사결정 지원] ← [L5 개인화 추천] ──┘
```

| Layer | 명칭 | 핵심 활동 | 주요 주체 |
|-------|------|----------|-----------|
| L1 | 상품 소싱·기획 | 아이템 발굴, 가격 설계, MD | 셀러, 브랜드 |
| L2 | 등록·콘텐츠 제작 | 사진·상세페이지·태그·키워드 | 셀러 |
| L3 | 검색·노출·랭킹 | 카탈로그, 검색엔진, 상품 매칭 | 플랫폼 |
| L4 | 광고·마케팅 | 키워드 광고, 프로모션, CRM | 셀러·플랫폼 |
| L5 | 개인화 추천 | 홈피드, 연관상품, 큐레이션 | 플랫폼 |
| L6 | 의사결정 지원 | 리뷰·Q&A·가격비교·시뮬레이션 | 소비자(플랫폼 제공) |
| L7 | 결제·금융 | PG, 할부, 후불결제(BNPL) | 플랫폼·금융사 |
| L8 | 재고·풀필먼트 | WMS, 입출고, 재고 예측 | 셀러·물류 |
| L9 | 배송·라스트마일 | 라우팅, 배송기사, 새벽·당일배송 | 물류 |
| L10 | CS·반품·교환 | 문의 응대, 환불, 회수 | 셀러·플랫폼 |
| L11 | 리뷰·재구매·로열티 | UGC, 멤버십, 재구매 사이클 | 소비자·플랫폼 |

## 3. 이해관계자 × 레이어 페인포인트 매트릭스

> 같은 레이어라도 **누구의 관점**에서 보느냐에 따라 문제의 성격이 다르다. 이 매트릭스가 문제 발굴의 좌표계.

| 이해관계자 | 주된 페인포인트 (관련 레이어) |
|-----------|----------------------------|
| **소비자** | 결정 피로(L5,L6) · 가짜리뷰/정보 비대칭(L6) · 사이즈/핏 미스매치(L6) · 반품 번거로움(L10) · 가격 변동·최저가 추적 피로(L4,L6) |
| **소상공인 셀러** | 상품 등록 시간 부담(L2) · 광고비 효율(L4) · 재고 예측·과다재고(L8) · 가짜리뷰·악성 리뷰 대응(L11) · CS 응대 인력 부족(L10) |
| **브랜드/대형 셀러** | 멀티채널 가격·재고 통합(L1,L8) · 콘텐츠 다국어/현지화(L2) · 마진율·프로모션 최적화(L4) · 반품률 관리(L10) |
| **플랫폼** | 콜드스타트 사용자 추천(L5) · 어뷰징·가짜리뷰 탐지(L11) · 검색 정확도(L3) · CS 봇·비용 절감(L10) · 카탈로그 정합성(L3) |
| **물류·배송** | 라우팅 최적화(L9) · 반품 역물류 비용(L9,L10) · 인력 효율(L9) |
| **결제·금융** | 부정거래 탐지(L7) · 신용평가/BNPL 리스크(L7) |

## 4. 문제 발굴 가이드 — 5가지 필터

아이디어 후보를 평가할 때 아래 질문을 통과해야 한다.

1. **명확한 페인** — 누가, 얼마나 자주, 얼마나 크게 아픈가? (정성·정량 모두)
2. **AI 결정력** — AI/ML이 _없으면 안 되는_ 영역인가? 단순 CRUD/룰 기반으로 풀리진 않는가?
3. **데이터 가용성** — 학생 팀이 합리적으로 확보·생성할 수 있는 데이터인가? (공개 데이터셋, 합성, 크롤링 윤리)
4. **차별화** — 기존 서비스(쿠팡·네이버·아마존) 대비 명확한 차별점이 있는가?
5. **데모 임팩트** — 발표·PT에서 _보여줄 수 있는_ 시각적·체감적 가치가 있는가?
