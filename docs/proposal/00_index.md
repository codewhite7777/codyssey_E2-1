## 3. 작성 원칙

1. **한 섹션 한 메시지** — 각 섹션은 한 줄 핵심 주장으로 요약 가능해야 한다.
2. **구체성** — 숫자·사례·인용으로 뒷받침. "많은 사용자" → "약 X만 명".
3. **일관성** — 섹션 간 사용자 정의·서비스명·기능명이 모두 동일해야 한다.
4. **시각화** — 표·다이어그램·플로우를 적극 사용 (텍스트 벽 금지).
