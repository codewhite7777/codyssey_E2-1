# 7. 향후 발전 방향

본 서비스는 MVP에서 검증 가능한 핵심 문제를 먼저 해결하고,  
이후 카테고리·기술·사업모델을 단계적으로 확장하는 전략을 따른다.  
초기에는 공유 시트 기반 광고 검증 도구로 시작하고,  
중기에는 가족·기관이 함께 쓰는 판단 보조 시스템,  
장기에는 플랫폼·기관용 광고 모니터링 인프라로 발전시키는 것을 목표로 한다.

---

## 7.1 단계별 로드맵

### 1단계 — MVP (0~3개월)

- 목표: 사용자가 숏폼 광고를 공유하면 60초 이내에 위험도와 근거를 보여주는 최소 기능 제품 구현
- 포함 기능: F1(광고 즉시 불러오기), F2(허위·과장 포인트 분석 — **딥페이크 판별 포함**), F3(근거 리포트 및 판단 지원). §4 기술 단위로는 OCR·VLM·Claim 추출·RAG·**MesoNet + Wav2Vec2 딥페이크 탐지**(F4) 전체를 MVP 범위로 확정.
- 완료 조건:
  - 라벨링된 광고 샘플 300건 이상 확보
  - 평균 분석 완료 시간 60초 이내
  - Claim 추출 F1 0.80 이상
  - 고위험 광고 탐지 정밀도 85% 이상
  - 파일럿 사용자 100명 이상 확보

### 2단계 — 확장 (3~9개월)

- 목표: 서비스 정확도와 재사용성을 높이고, 가족 공유·운영자 검토 기능 추가
- 추가 기능:
  - 분석 결과 저장 및 히스토리 조회
  - 가족·지인 공유 기능
  - 사용자 피드백 수집 및 재학습 루프
  - 운영자용 검토 대시보드
  - 카테고리별 위험 패턴 세분화
  - **딥페이크 판별(F4) 고도화** — 립싱크 미세 오차 분석·다중 팩터 융합 스코어링
- 완료 조건:
  - 누적 광고 데이터 2,000건 이상
  - 월간 활성 사용자(MAU) 1,000명 이상
  - 평균 분석 완료 시간 40초 이내
  - 고위험 광고 탐지 정밀도 88% 이상
  - 사용자 재사용률 35% 이상

### 3단계 — 고도화 (9~18개월)

- 목표: 소비자용 앱을 넘어 기관·플랫폼이 활용 가능한 광고 모니터링 보조 체계로 확장
- 추가 기능:
  - 멀티모달 검색 고도화
  - 광고 위험 패턴 자동 군집화
  - 기관·플랫폼용 대시보드
  - 외부 API 제공
  - 카테고리·국가별 정책 룰셋 분리 운영
- 완료 조건:
  - 누적 광고 데이터 10,000건 이상
  - 월간 활성 사용자(MAU) 5,000~10,000명
  - 평균 분석 완료 시간 30초 이내
  - 고위험 광고 탐지 정밀도 90% 이상
  - 외부 기관·플랫폼 PoC 2건 이상

## 7.2 확장 시나리오

### 인접 도메인으로의 확장

초기에는 건강기능식품·화장품·다이어트 광고에 집중한다.  
이후 같은 기술 자산을 활용해 육아·반려동물·생활용품·일반 식품 광고 검증으로 확장할 수 있다.  
장기적으로는 가짜 후기·과장 비교·가격·긴급성 조작이 잦은 다른 소비재 카테고리에도 적용 가능하다.

### 카테고리·국가 확장 순서

| 단계 | 카테고리 | 언어·국가 |
|------|----------|-----------|
| 1차 | 건강기능식품 / 화장품 / 다이어트 | 한국어 광고 |
| 2차 | 일반 식품 / 생활용품 / 육아용품 | 한국어+영어 혼합 광고 |
| 3차 | 반려동물 / 시니어 소비재 / 해외 직구 | 일본·동남아 등 규정 현지화 가능 국가 |

### 파트너십·B2B 모델

소비자 대상 앱을 기반으로 다음과 같은 B2B 확장이 가능하다.

- 플랫폼 대상: 위험 광고 사전 탐지 보조
- 규제기관·소비자보호기관 대상: 모니터링 대시보드
- 커머스 플랫폼 대상: 판매자 광고 사전 점검 도구

목표: 1년 내 PoC 2건 이상 / 18개월 내 기관·플랫폼용 유료 계약 1건 이상

## 7.3 기술 발전 로드맵

| 영역 | 현재 (MVP) | 중기 | 장기 |
|------|-----------|------|------|
| 모델 | OCR + VLM + Claim 추출 + 규정 매핑 + **딥페이크 판별(MesoNet + Wav2Vec2)** | 하이브리드 검색·고위험 패턴 분류기·사용자 피드백 반영·**딥페이크 고도화(립싱크·다중 팩터 융합)** | 멀티모달 임베딩·자동 군집화·도메인 특화 탐지기 |
| 데이터 | 수동 샘플링 + 소규모 라벨링 (300건) | 반자동 라벨링 + 운영 데이터 축적 (2,000건) | 대규모 광고 데이터셋 + 규정·사례 지식베이스 (10,000건) |
| 인프라 | FastAPI + PostgreSQL + pgvector + Temporal | 워크플로 고도화·관리자 대시보드·검색 최적화 | API 상품화·기관용 대시보드·확장형 분석 인프라 |

**기술 발전 목표치 요약**

| 지표 | MVP | 중기 | 장기 |
|------|-----|------|------|
| 광고 데이터셋 규모 | 300건 | 2,000건 | 10,000건 |
| Claim 추출 F1 | 0.80 | 0.85 | 0.90 |
| 평균 분석 시간 | 60초 | 40초 | 30초 |
| 고위험 광고 탐지 정밀도 | 85% | 88% | 90% |

## 7.4 위험 요소 및 대응

| 위험 | 영향 | 대응 전략 |
|------|------|-----------|
| 플랫폼 정책 변경 | 광고 공유·수집 방식 제한 가능성 | 공식 공유 인터페이스 중심 설계, 플랫폼 정책 주기적 점검 |
| 규정 변화 속도 | 판정 기준이 빠르게 바뀔 수 있음 | 정책 DB·룰셋 주 1회 이상 갱신 |
| 데이터 부족 | 특정 위반 유형 학습 데이터 부족 | 합성 데이터 + 사용자 피드백 + 반자동 라벨링 활용 |
| 오탐·미탐 누적 | 사용자 신뢰 하락 가능성 | 위험도 단계 분리, 피드백 루프, hard case 재검토 |
| 처리 비용 증가 | 사용량 증가 시 모델 호출 비용 상승 | 경량 모델 우선, 고난도 샘플만 고성능 모델 fallback |
| 과도한 카테고리 확장 | 너무 많은 카테고리를 한 번에 다루면 품질 저하 | 1차 카테고리 집중 후 순차 확장 |
