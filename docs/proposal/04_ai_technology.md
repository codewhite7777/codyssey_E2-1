# 4. AI 기술 적용 방안

AI가 이 서비스의 핵심인 이유는 단순 키워드 매칭으로는 풀 수 없기 때문이다.  
"2주 만에 효과"라는 문구가 위반인지 아닌지는 카테고리·제품 유형·함께 나오는 장면(흰 가운, Before/After)과 **화면 속 인물이 실제 전문가인지 AI 합성 인물인지**를 동시에 봐야 판단할 수 있다.  
OCR·VLM·LLM·규정 엔진·딥페이크 판별기를 조합해 **멀티모달 분석을 MVP 단계부터 수행**하는 구조가 필요한 이유다.

## 4.1 기능–기술 매핑

| # | 기능 | 적용 AI 기술 | 모델/알고리즘 후보 | 입력 → 출력 |
|---|------|--------------|--------------------|-------------|
| F1 | 광고 분해 및 장면·문구 추출 | Video Understanding · **VLM 통합 OCR** · Scene Segmentation | **Gemini 2.5 Flash (주력 VLM·OCR 통합)**, Qwen2.5-VL-7B (자가호스팅 대안), **PP-OCRv5 (한국어 소형 자막·약관 보조 OCR)**, **PySceneDetect (장면 분할 전처리)** | 광고 URL·영상·썸네일·설명란 → 장면별 타임스탬프·키프레임, 자막·배너 문구, 시각 단서(흰 가운·Before/After·카운트다운 등) |
| F2 | Claim 추출 및 위반 유형 분류 | Structured Extraction, Claim Detection, Policy-as-Code | Gemini Structured Output, OPA, BGE-M3 + pgvector | OCR 텍스트·설명란·랜딩페이지·규정 문서 → claim JSON, 위반 유형, 근거 문서 후보 |
| F3 | 교차검증 및 설명 가능한 리포트 생성 | Cross-modal Retrieval, RAG, Risk Scoring | Gemini 2.5 Flash/Pro, BGE-M3 + rerank, pgvector | 장면·문구·claim·근거 문서 → 위험도, 핵심 의심 포인트, 사용자용 리포트 |
| F4 | AI 합성 인물(딥페이크) 판별 | **Lightweight CNN (얼굴 위조 탐지)**, **Audio Anti-Spoofing (음성 합성 탐지)** | **MesoNet (MVP 기준선) → LAA-Net / EfficientNet-B7 (Phase 2 업그레이드)**, **AASIST (음성 판별기) + Wav2Vec2 (feature extractor)** | 광고 영상 프레임·음성 → AI 합성 확률, 위조 흔적 리포트 |

> **F1~F4 모두 MVP에서 구현된다.** §1.1.4 의 AI 가짜 전문가 4건 실사례 및 식약처 AI 부당광고 **63건 적발(2025)** 이 본 프로젝트의 핵심 근거이므로, 딥페이크 판별을 Phase 2로 미루면 문제 정의 대비 해법 범위가 좁아진다는 판단이다. 다만 **립싱크 미세 오차 분석·고차원 다중 팩터 융합 스코어링** 은 Phase 2 에서 고도화한다.

## 4.2 기술 선택 이유

### F1 — 광고 분해 및 장면·문구 추출

숏폼 광고는 흰 가운·병원 배경·할인 배지 같은 장면 정보와 "2주 만에"·"완치"·"오늘만" 같은 문구 정보가 함께 결합되어 효과를 낸다. 텍스트만 보거나 영상만 보는 단일 모델로는 이 맥락을 잡기 어렵다.

- **주력: VLM (Gemini 2.5 Flash / Qwen2.5-VL-7B)** — 최신 VLM 은 **장면 해석 + 화면 내 텍스트 인식(OCR)을 동시 수행** 가능. 프롬프트에서 *"화면 내 모든 텍스트 추출 + 장면 의미·연출 요소 해석"* 을 지시해 **한 번의 호출로 통합 분석** → 비용·지연 이중화 방지.
- **보조 OCR: PP-OCRv5** — VLM 이 놓치기 쉬운 **소형 자막·약관 고지·한국어 특수 폰트**(예: 빠르게 스크롤되는 효능 주장 자막)에 한해 보조 호출. 한국어 인식 정확도 보완.
- **Fallback: Gemini 2.5 Pro** — VLM 신뢰도 낮을 때만 호출하는 고성능 대안.

> **영상 전처리 — PySceneDetect 장면 분할 + 프레임 샘플링**: 15~60초 숏폼 영상을 **PySceneDetect** 로 **장면 전환 기반 분할** → 각 장면 대표 키프레임 추출 + 1초 균등 샘플링 병행. 평균 **20~30 프레임** 을 분석 대상으로 삼으며, **장면별 타임스탬프** 를 기록해 *"영상 0:12 ~ 0:18 구간에서 '완치' 주장이 등장"* 식으로 리포트에 구간 단위 근거를 명시할 수 있다.

### F2 — Claim 추출 및 위반 유형 분류

자유 요약 방식은 결과가 매번 달라져 규정 매핑과 설명이 불안정해진다. Structured Output으로 뽑으면 claim_text·claim_type·evidence_span·policy_candidate 같은 필드를 일관되게 추출할 수 있다. 본 서비스는 **LLM 1차 판정 (자연어 해석) + Rule Engine 2차 필터 (결정론적 키워드·카테고리 매칭)** 하이브리드 구조로 설계한다.

- **Claim 추출 (1차)**: Gemini Structured Outputs (MVP 주력) — **API 비용·지연·개인정보 외부 전송 우려 시 자가호스팅 대안 Qwen2.5-Instruct / Llama-3-Korean fine-tuning 으로 전환 가능** (하이브리드 구조: 경량 내부 모델 선행 + 어려운 샘플만 Gemini fallback)
- **LLM 1차 판정**: 추출된 claim 을 LLM 이 *"의학적 효능 암시 / 과장 표현 / 권위 연출 / 긴급성 조작"* 등 위반 유형으로 분류. **자연어 해석이 본질인 판단은 LLM 담당**.
- **Rule Engine 2차 필터 (OPA)**: LLM 판정 위에 **결정론적 규칙** 만 추가 — *명시적 금지 키워드* (*"완치·특효·100%"*)·*카테고리 고정 규칙* (건기식 → 의약 효능 표방 금지)·*수치 단정 표현* 등. OPA 는 *"불확실한 자연어 해석"* 이 아닌 *"명확한 키워드·카테고리 매칭"* 전용으로 역할 한정해 **Rego 룰셋 폭발을 방지**.
- **근거 검색**: BGE-M3 + pgvector (하이브리드 검색, MVP 현실적 선택) — 관련 규정 조항을 벡터 검색으로 찾아 LLM·OPA 판정 근거로 제시.

**역할 분리의 핵심**: LLM 은 *"왜 의심스러운가"* 의 자연어 해석, OPA 는 *"규정에 명시된 금지 사항에 해당하는가"* 의 결정론적 판정. 규정이 바뀌어도 모델 재학습 없이 룰셋만 업데이트 가능하면서, OPA 룰이 자연어까지 다루지 않아 복잡도 관리 가능한 수준으로 유지됨.

> **OPA 룰셋 작성 프로세스**: 식약처·공정위·표시광고법 조항 중 **결정론적 규칙으로 표현 가능한 것**(금지 키워드·카테고리 제한·수치 단정 등)만 Rego 언어로 변환한다. 작업은 **법률 자문 또는 팀 내 규정 담당자가 수행**하며, 기획·개발 팀 사전 검토를 거친다. 규정이 변경되면 동일 프로세스로 **룰셋만 갱신** — 모델 재학습 불필요. **자연어 해석이 필요한 조항은 LLM 담당으로 분리**해 Rego 룰셋 복잡도를 관리 가능한 수준으로 유지한다.

### F3 — 교차검증 및 리포트 생성

단순히 "위험"이라고만 하면 사용자가 신뢰하기 어렵다. 광고 장면·claim·랜딩페이지·규정 문서를 대조해 "어느 문구가 어떤 조항에 해당하는지"를 근거와 함께 제시해야 한다.

- MVP: BGE-M3 + 하이브리드 검색 (텍스트 중심 RAG)
- 확장: Gemini Embedding 2 (멀티모달 임베딩으로 전환)

### F4 — AI 합성 인물(딥페이크) 판별 ★ MVP 포함

§1.1.4 에서 확인한 대로 **AI 생성 '가짜 전문가' 부당광고가 식약처에만 2025년 63건 적발**됐고[^1], 본 기획서 §1.1.4 에는 유튜브 Shorts 유통 AI 딥페이크 의사 광고 **4건 실사례 스크린샷** 이 수록돼 있다. 주 타겟 페르소나 박영수(68세, §2.2)가 겪는 "AI 합성 의사" 문제가 정확히 이 카테고리이므로, **딥페이크 판별 없이 MVP를 출시하면 문제 정의와 해법 범위가 불일치**한다.

**영상 딥페이크 (얼굴 위조 탐지)**

- **MVP 기준선: MesoNet** — 약 30K 파라미터 경량 CNN. 프레임 텍스처·압축 아티팩트 기반 탐지. 학생 팀 GPU 리소스로 fine-tuning·추론 모두 경량. *2018년 모델로 **최신 생성 기술(SDXL·Sora·Runway Gen-3·최신 FaceFusion)에 대한 대응력은 제한적***.
- **Phase 2 권장 업그레이드**: **LAA-Net** (Localized Artifact Attention Network, 2024) 또는 **EfficientNet-B7 앙상블** — 최신 생성 모델 detection 성능이 MesoNet 대비 크게 향상. 리소스 여유 시 MVP 단계부터 병행 도입도 검토.
- *(참고)* **Intel FakeCatcher** (rPPG 기반 혈류 신호 분석) — 상용이지만 기술 논문 공개. 계절별 검증용으로 활용 가능.

**음성 딥페이크 (음성 합성·변조 탐지)** — *Anti-Spoofing 이 정확한 범주*

- **MVP 주력: AASIST** (Interspeech 2022) — ASVspoof 챌린지 상위 **음성 anti-spoofing 전용 분류기**. Spectro-temporal 그래프 어텐션 기반.
- **대안: RawNet2** — raw waveform 직접 처리, AASIST 와 함께 ASVspoof 상위권.
- **Wav2Vec2 는 판별기(classifier)가 아니라 feature extractor 로만 사용** — pretrain 된 Wav2Vec2 의 음성 임베딩을 앞단에, 뒷단에 ASVspoof 로 fine-tuning 한 **분류기 헤드(classification head)** 를 결합하는 표준 구조. Wav2Vec2 단독으로는 딥페이크 판별 기능을 수행하지 않음에 주의.

> **⚠️ 음성 ASR 범위와의 구분**: 본 F4 의 음성 포렌식(AASIST/RawNet2)은 *"음성이 합성되었는가"* 를 판별하는 anti-spoofing 이며, *"음성이 무슨 말을 하는가"* 를 텍스트화하는 ASR(Whisper 등)은 §1.6 MVP 범위에서 1차 제외다. 음성 속 **주장 내용 검증**은 자막(보조 OCR) · VLM 간접 분석으로 커버하며, Whisper 기반 완전 ASR 은 **Phase 2 편입 검토** 항목으로 둔다.

**융합 스코어링** — F4(딥페이크)·F1(영상 단서)·F2(효능 문구) 결과를 한 리포트에서 교차 노출해 *"S대 전문의 자칭 + 흰 가운 + AI 합성 얼굴"* 같은 **결합 기만 패턴** 을 사용자에게 직관적으로 보여준다.

사전학습 모델 + 경량 fine-tuning 으로 MVP 목표치 달성이 가능하며, Phase 2 고도화 단계에서는 **LAA-Net (영상) · AASIST 고성능 변종 (음성) · 립싱크 미세 오차 분석 · 고차원 다중 팩터 융합** 을 도입해 최신 딥페이크 기법에도 방어력을 확보한다.

### Fusion Scorer — 다중 분석 결과 통합 가중치

F1~F4 의 개별 분석 결과(효능 과장·권위 연출·AI 합성 여부 등)를 **단일 위험도 점수** 로 통합해 사용자 리포트에 제공한다.

- **MVP 초기**: 위반 유형별 가중치를 **수동 설정** (예: 딥페이크 50% + 효능 과장 30% + 권위 연출 20%). 법적 위험도·사회적 피해 규모를 기반으로 팀이 합의해 결정.
- **Phase 2**: 사용자 피드백(오탐·미탐 신고·리포트 만족도) 기반으로 **Logistic Regression 또는 LightGBM 가중치 학습** — 오탐 감소·정밀도 향상.
- **Explainability**: 최종 점수가 어떤 요소로 구성되었는지 리포트에 **명시적으로 분해 표시** (예: *"딥페이크 의심 0.80 + 효능 단정 문구 0.70 → 종합 위험도 고위험"*) — 사용자가 **왜 이 판정인가**를 직접 확인 가능.

## 4.3 데이터 전략

### 데이터 소스

**규정·정책 문서 (판정 기준선)**
표시광고법 관련 지침, 식약처 부당광고 사례집, 공정위 의결서, 플랫폼 광고 정책, 허용·금지 표현 사전. 이 서비스는 규제 민감 카테고리에서의 근거 기반 판정이 핵심이므로, 정책 문서 자체가 모델 학습보다 더 중요한 자산이다.

**실제 광고 샘플**
YouTube Shorts·Instagram Reels·TikTok에서 팀이 직접 샘플링한 광고, 사용자 공유로 들어온 광고(동의 기반), 공개적으로 접근 가능한 광고성 게시물 및 랜딩페이지.

**합성 데이터**
특정 위반 유형(전문의 권위 연출·오늘만 특가·비포/애프터·과장 수치)은 실제 표본이 적을 수 있다. 텍스트 패턴과 디자인 패턴을 바꿔가며 합성 광고 샘플을 생성해 희귀 사례 학습과 테스트셋 보강에 활용한다.

**공개 데이터셋**
- AI Hub: 국내 특화 허위·과장 광고 텍스트·이미지 데이터 (F1~F3)
- **FaceForensics++** : 딥페이크 판별용 영상 데이터셋 — **F4 MVP 학습·평가**[^2]
- **ASVspoof 2019/2021** : 음성 스푸핑 공개 데이터셋 (F4 Wav2Vec2 fine-tuning 후보)

| **딥페이크 라벨** | **AI 합성 여부·얼굴 합성 기법(단일 프레임 교체/전체 얼굴 생성/립싱크)·음성 합성 여부** |
| 근거 라벨 | 관련 규정 문서 ID, 근거 문장 span |
| 판정 라벨 | 정상 / 주의 / 고위험 / 추가 검토 |

### 데이터 품질 관리

학습/검증/테스트 분할은 프레임 단위가 아니라 광고 캠페인·브랜드 ID 단위로 한다. 같은 광고의 인접 프레임이 학습셋과 테스트셋에 동시에 들어가면 성능이 과대평가된다.

균형 기준: 플랫폼(Shorts/Reels/TikTok), 카테고리(건기식/화장품/다이어트), 위반 유형(효능/권위/가격/긴급성/후기) 균등 유지. Hard negative set(합법이지만 과격하게 보이는 광고) 별도 유지. Annotator agreement는 Cohen's kappa로 추적.

**딥페이크 도메인 갭 관리** — FaceForensics++ 같은 공개 데이터는 도메인(뉴스·정치인 얼굴)이 본 서비스 타겟(가상 전문의·의료 세트)과 다르므로, 실제 광고 샘플 기반 fine-tuning + 합성 광고(흰 가운·병원 배경 포함)로 도메인 적응 필수.

## 4.4 모델 평가 지표

| 모델/서브시스템 | 1차 지표 | 2차 지표 | MVP 목표치 |
|----------------|----------|----------|------------|
| OCR | CER (Character Error Rate) | Overlay text recall | CER 0.12 이하 |
| Claim 추출 | Span/Entity F1 | JSON schema validity | F1 0.80 이상, schema valid 99% 이상 |
| 위반 유형 분류 | Macro F1 | 고위험 Precision | Macro F1 0.80 이상, 고위험 Precision 0.85 이상 |
| 근거 회수 | Recall@5 | MRR@10 | R@5 0.90 이상 |
| **딥페이크 판별 (F4)** | **AUC-ROC** | **고위험 판정 Precision / FPR @ 90% Recall** | **AUC 0.85 이상 · 고위험 Precision 0.90 이상** |
| 최종 리포트 | 사용자 긍정 평가율 | 설명 만족도 | 80% 이상 긍정 |

## 4.5 운영·서빙 전략

- 추론 환경: 비동기 처리 기본. 경량 모델 우선 호출, 고난도 샘플만 고성능 모델 fallback.
- **F4 2-tier 호출** — MesoNet 우선 호출 → 의심 점수가 임계값 초과한 경우에만 Wav2Vec2 음성 분석 병행. MesoNet 추론 비용은 수십 KB 파라미터 수준으로 **전체 비용에 거의 영향 없음**.
- 모델 갱신 주기: 정책 문서·룰셋 주 1회 이상 / 프롬프트·extraction schema 수시 / 경량 분류기·**F4 딥페이크 탐지기** 월 1회 이상 재학습.
- 모니터링: 오탐·미탐 비율 주기적 추적, 사용자 피드백 기반 hard case 재검토.
- 처리 비용 관리: 사용량 증가 시 모델 호출 비용 상승 대응을 위해 경량 모델 우선 원칙 유지.
