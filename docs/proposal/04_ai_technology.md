# 4. AI 기술 적용 방안

AI가 이 서비스의 핵심인 이유는 단순 키워드 매칭으로는 풀 수 없기 때문이다.  
"2주 만에 효과"라는 문구가 위반인지 아닌지는 카테고리·제품 유형·함께 나오는 장면(흰 가운, Before/After)을 동시에 봐야 판단할 수 있다.  
OCR·VLM·LLM·규정 엔진을 조합해 멀티모달 분석을 수행하는 구조가 필요한 이유다.

---

## 4.1 기능–기술 매핑

| # | 기능 | 적용 AI 기술 | 모델/알고리즘 후보 | 입력 → 출력 |
|---|------|--------------|--------------------|-------------|
| F1 | 광고 분해 및 장면·문구 추출 | OCR, Video Understanding, Scene Segmentation | PP-OCRv5 (MVP), Gemini 2.5 Flash, Qwen2.5-VL-7B (자가호스팅 대안) | 광고 URL·영상·썸네일·설명란 → 장면 요약, 프레임별 자막, 시각 단서(흰 가운·Before/After·카운트다운 등) |
| F2 | Claim 추출 및 위반 유형 분류 | Structured Extraction, Claim Detection, Policy-as-Code | Gemini Structured Output, OPA, BGE-M3 + pgvector | OCR 텍스트·설명란·랜딩페이지·규정 문서 → claim JSON, 위반 유형, 근거 문서 후보 |
| F3 | 교차검증 및 설명 가능한 리포트 생성 | Cross-modal Retrieval, RAG, Risk Scoring | Gemini 2.5 Flash/Pro, BGE-M3 + rerank, pgvector | 장면·문구·claim·근거 문서 → 위험도, 핵심 의심 포인트, 사용자용 리포트 |
| F4 | AI 합성 인물(딥페이크) 판별 | Vision Transformer, 음성 변조 분석 | MesoNet, Wav2Vec2 | 광고 영상 프레임·음성 → AI 합성 확률, 위조 흔적 리포트 |

> F4(딥페이크 판별)는 2단계 도입 예정이다. MVP에서는 F1~F3를 우선 구현한다.

## 4.2 기술 선택 이유

### F1 — 광고 분해 및 장면·문구 추출

숏폼 광고는 흰 가운·병원 배경·할인 배지 같은 장면 정보와 "2주 만에"·"완치"·"오늘만" 같은 문구 정보가 함께 결합되어 효과를 낸다. 텍스트만 보거나 영상만 보는 단일 모델로는 이 맥락을 잡기 어렵다.

- MVP: PP-OCRv5 (자막·배너 문구 추출) + Gemini 2.5 Flash (장면 의미·연출 요소 해석)
- 어려운 샘플: Gemini 2.5 Pro
- 자가호스팅 대안: Qwen2.5-VL-7B

OCR은 화면 자막과 배너 문구를 읽는 데 강하고, VLM은 장면 의미를 해석하는 데 강하다. 둘을 함께 써야 광고를 제대로 분해할 수 있다.

### F2 — Claim 추출 및 위반 유형 분류

자유 요약 방식은 결과가 매번 달라져 규정 매핑과 설명이 불안정해진다. Structured Output으로 뽑으면 claim_text·claim_type·evidence_span·policy_candidate 같은 필드를 일관되게 추출할 수 있다.

- Claim 추출: Gemini Structured Outputs (LLM이 추출, 구조화)
- 정책 판정: OPA (Open Policy Agent) — 규칙을 코드로 분리해 관리, 갱신 용이
- 근거 검색: BGE-M3 + pgvector (하이브리드 검색, MVP 현실적 선택)

LLM은 추출, OPA는 판정으로 역할을 분리하는 것이 핵심이다. 규정이 바뀌었을 때 모델 재학습 없이 룰셋만 업데이트하면 되기 때문이다.

### F3 — 교차검증 및 리포트 생성

단순히 "위험"이라고만 하면 사용자가 신뢰하기 어렵다. 광고 장면·claim·랜딩페이지·규정 문서를 대조해 "어느 문구가 어떤 조항에 해당하는지"를 근거와 함께 제시해야 한다.

- MVP: BGE-M3 + 하이브리드 검색 (텍스트 중심 RAG)
- 확장: Gemini Embedding 2 (멀티모달 임베딩으로 전환)

### F4 — 딥페이크 판별 (2단계)

식약처가 2025년 한 해 동안 AI 생성 '가짜 전문가' 활용 부당광고를 63건 적발했다.[^1] 단순 외형 분석을 넘어 음성의 비자연적 주파수 특성과 립싱크 오차를 함께 분석하는 멀티 팩터 검증이 필요하다.

- MesoNet: 영상 프레임의 텍스처 변화·압축 아티팩트 분석
- Wav2Vec2: 음성 합성 흔적 감지

## 4.3 데이터 전략

### 데이터 소스

**규정·정책 문서 (판정 기준선)**
표시광고법 관련 지침, 식약처 부당광고 사례집, 공정위 의결서, 플랫폼 광고 정책, 허용·금지 표현 사전. 이 서비스는 규제 민감 카테고리에서의 근거 기반 판정이 핵심이므로, 정책 문서 자체가 모델 학습보다 더 중요한 자산이다.

**실제 광고 샘플**
YouTube Shorts·Instagram Reels·TikTok에서 팀이 직접 샘플링한 광고, 사용자 공유로 들어온 광고(동의 기반), 공개적으로 접근 가능한 광고성 게시물 및 랜딩페이지.

**합성 데이터**
특정 위반 유형(전문의 권위 연출·오늘만 특가·비포/애프터·과장 수치)은 실제 표본이 적을 수 있다. 텍스트 패턴과 디자인 패턴을 바꿔가며 합성 광고 샘플을 생성해 희귀 사례 학습과 테스트셋 보강에 활용한다.

**공개 데이터셋**
- AI Hub: 국내 특화 허위·과장 광고 텍스트·이미지 데이터
- FaceForensics++: 딥페이크 판별용 영상 데이터셋 (F4 도입 시)[^2]

### 라벨링 단위 및 항목

라벨링 단위는 프레임이 아니라 광고 단위 → 장면 단위 → claim 단위 순으로 설정한다.

| 라벨 종류 | 세부 항목 |
|-----------|-----------|
| 광고 메타 | 플랫폼, 카테고리, 링크 유형, 영상 길이 |
| 시각 라벨 | 흰 가운·병원 배경·Before/After·할인 배지·카운트다운 |
| 텍스트 라벨 | claim text, 수치 표현, 기간 표현, 권위 표현 |
| 위반 라벨 | 효능 과장·가격 허위·긴급성 조작·가상인물 의심·후기 오인 가능성 |
| 근거 라벨 | 관련 규정 문서 ID, 근거 문장 span |
| 판정 라벨 | 정상 / 주의 / 고위험 / 추가 검토 |

### 데이터 품질 관리

학습/검증/테스트 분할은 프레임 단위가 아니라 광고 캠페인·브랜드 ID 단위로 한다. 같은 광고의 인접 프레임이 학습셋과 테스트셋에 동시에 들어가면 성능이 과대평가된다.

균형 기준: 플랫폼(Shorts/Reels/TikTok), 카테고리(건기식/화장품/다이어트), 위반 유형(효능/권위/가격/긴급성/후기) 균등 유지. Hard negative set(합법이지만 과격하게 보이는 광고) 별도 유지. Annotator agreement는 Cohen's kappa로 추적.

## 4.4 모델 평가 지표

| 모델/서브시스템 | 1차 지표 | 2차 지표 | MVP 목표치 |
|----------------|----------|----------|------------|
| OCR | CER (Character Error Rate) | Overlay text recall | CER 0.12 이하 |
| Claim 추출 | Span/Entity F1 | JSON schema validity | F1 0.80 이상, schema valid 99% 이상 |
| 위반 유형 분류 | Macro F1 | 고위험 Precision | Macro F1 0.80 이상, 고위험 Precision 0.85 이상 |
| 근거 회수 | Recall@5 | MRR@10 | R@5 0.90 이상 |
| 최종 리포트 | 사용자 긍정 평가율 | 설명 만족도 | 80% 이상 긍정 |

## 4.5 운영·서빙 전략

- 추론 환경: 비동기 처리 기본. 경량 모델 우선 호출, 고난도 샘플만 고성능 모델 fallback.
- 모델 갱신 주기: 정책 문서·룰셋 주 1회 이상 / 프롬프트·extraction schema 수시 / 경량 분류기 월 1회 이상.
- 모니터링: 오탐·미탐 비율 주기적 추적, 사용자 피드백 기반 hard case 재검토.
- 처리 비용 관리: 사용량 증가 시 모델 호출 비용 상승 대응을 위해 경량 모델 우선 원칙 유지.
