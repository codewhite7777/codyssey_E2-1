## 3.2 한 줄 정의 (Elevator Pitch)

> 숏폼에서 본 건강·뷰티 광고를 공유하면, AI가 허위·과장 가능성과 근거를 1분 이내에 보여주는 구매 전 검증 서비스다.

---

## 3.3 핵심 기능 (Core Features)

| # | 기능 | 설명 | 사용자 가치 |
|---|------|------|------------|
| F1 | 광고 즉시 불러오기 | 사용자가 YouTube Shorts·Instagram Reels·TikTok 등에서 의심되는 광고를 공유 버튼으로 앱에 전달하면, 영상·자막·링크·설명란을 수집해 분석을 시작한다. | 복사·붙여넣기, 재검색, 스크린샷 없이 의심한 순간 바로 검증할 수 있다. |
| F2 | 허위·과장 포인트 분석 | AI가 광고의 영상 장면·자막·권위 연출·효능 문구·전후 비교·긴급성 표현 등을 분석해 문제 가능성이 높은 부분을 추출한다. AI 합성 가상 인물(딥페이크) 여부도 판별한다. | "이 광고가 이상하다"는 막연한 느낌을 넘어, 어느 부분이 왜 의심스러운지 구체적으로 이해할 수 있다. |
| F3 | 근거 리포트 및 판단 지원 | 분석 결과를 '주의'·'보류'·'추가 검증 필요' 등으로 정리하고, 관련 근거(규정·법령)와 함께 제공한다. 가족에게 공유하거나 나중에 다시 확인할 수 있다. | 충동 구매를 줄이고, 구매 전 한 번 더 생각할 수 있는 판단 근거를 얻는다. |

> MVP 호출 방식은 '공유 시트(Share Sheet) 수동 호출'을 기본으로 한다.
> 능동형 실시간 감지(Accessibility Service 기반 오버레이)는 플랫폼·OS 정책 제약을 고려해 2단계 이후 도입을 검토한다.

## 3.4 핵심 사용 시나리오 (Key User Flow)

```
사용자 진입
  → 숏폼 광고 시청 중 의심 발생
  → 공유 버튼으로 앱 호출
  → AI가 영상·자막·주장 분석
  → 위험도·의심 포인트·근거 리포트 제공
  → 구매 보류 / 추가 검증 / 가족 공유
```

### 시나리오 예시 — 이정희 씨 (주 타겟)

1. 유튜브 Shorts에서 "전문의 추천 관절 영양제 / 2주 만에 효과" 광고 시청.
2. 공유 버튼 → 앱 선택.
3. 앱이 "권위 연출 사용 / 효능 단정 표현 / 전후 비교 연출" 리포트 제공.
4. 이정희 씨는 광고를 감이 아닌 근거로 판단, 구매 보류 또는 추가 검색 결정.

### 시나리오 예시 — 김서연 씨 (부 타겟)

1. 부모님이 카카오톡으로 보낸 건강상품 광고 링크 수신.
2. 같은 방식으로 앱에 전달, 분석 결과 수신.
3. 리포트를 부모님께 공유하며 구매 재고 유도.

## 3.5 차별점

### 기존 서비스 대비

| 항목 | 기존 방식 | 본 서비스 |
|------|-----------|-----------|
| 문제 해결 시점 | 광고를 본 뒤 사용자가 직접 검색·비교·후기 탐색 | 광고를 본 그 순간, 공유 한 번으로 분석 결과 제공 |
| 분석 대상 | 광고주 정보·웹사이트 신뢰도·신고 기능 위주 | 숏폼 영상 자체의 자막·주장·권위 연출·효능 과장 직접 분석 |
| 분석 형태 | 텍스트 검색 기반, 사용자가 직접 판단 | 멀티모달 AI 분석 + 규정 근거 포함 리포트 자동 생성 |

### 경쟁 서비스 비교

|--------|------|------|-------------------|

| Meta Ad Library | Meta 계열 광고 검색·확인 가능 | 광고 내용 자체의 위험성 분석 없음 | 광고 내용 해석·의심 포인트 요약 제공 |
