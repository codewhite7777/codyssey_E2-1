# 3. 서비스 개요

> **작성 가이드**: 본 서비스는 사용자가 직접 검증을 요청하는 '수동형 호출'뿐만 아니라,\
>  위험 광고 노출 시 즉각 경고를 띄우는 **'능동형 호출'** 기술을 통해 전방위적인 안전망을 구축한다.\
아직 작성이 초안이기 때문에 충분히 변할 수 있음을 인지하자! (아직 정해지지 않은 서비스명이 변할 수 있다..)

---

## 3.1 서비스 이름
(본 서비스 이름 및 유래, 로고 및 심볼 컨셉들은 아직 정해지지 않아서 변할 수 있다!) 
- **정식 명칭**: **Ad-Guardian AI (애드가디언 AI)**
- **이름의 의미·유래**: 
    - **Ad(광고)** + **Guardian(수호자)**의 합성어. 기만적인 광고로부터 사용자를 실시간으로 보호(Guard)한다는 의미.
- **로고·심볼 컨셉**: 
    - 숏폼 피드 위에 떠 있는 **'신뢰의 방패'**. 
    - 평상시에는 투명한 심볼로 존재하다가, 위험 감지 시 **Amber Red**로 점멸하며 주의를 환기함.

## 3.2 한 줄 정의 (Elevator Pitch)

> "우리는 **숏폼 쇼핑 이용자**를 위해 **멀티모달 AI**를 활용하여, 별도의 조작 없이도 허위·과장 광고를 **자동 감지하여 경고**하는 **지능형 광고 검증 솔루션**입니다."

## 3.3 핵심 기능 (Core Features)

| # | 기능 | 설명 | 사용자 가치 |
|---|------|------|------------|
| **F1** | **능동형 실시간 스캐닝 (Active)** | 앱 실행 중 백그라운드에서 현재 시청 중인 영상의 오디오/자막을 분석하여 위반 소지 발견 시 상단 오버레이 알림을 띄운다. | 사용자가 인지하지 못한 위험(예: 알부민 과장 광고)을 선제적으로 차단 |
| **F2** | **멀티모달 딥페이크 판별** | 영상 속 인물의 얼굴 왜곡, 음성 합성 흔적을 추적하여 AI 가상 인물이나 사칭 의사 여부를 3초 내에 판별한다. | 가짜 전문가의 권위에 속아 발생하는 고가의 사기 구매 예방 |
| **F3** | **정밀 검증 온디맨드 (Passive)** | 사용자가 더 자세한 분석을 원할 경우, '공유하기' 시트를 통해 상세 리포트(위반 법령, 실제 성분 정보 등)를 즉시 호출한다. | 충동구매 직전 이성적인 판단을 도와주는 근거 데이터 제공 |

## 3.4 핵심 사용 시나리오 (Key User Flow)

### [능동형 호출 시나리오 - 박영수 씨 모델]
1. 유튜브 쇼츠를 시청 중 의사 사칭 '알부민' 광고 노출.
2. **Ad-Guardian AI**가 백그라운드에서 오디오의 '간 기능 개선', '특효' 키워드와 AI 합성 음성을 실시간 감지.
3. 화면 상단에 **[주의!: 허위 광고 확률 92%]** 팝업 노출 및 진동 알림.
4. 사용자가 시청을 중단하고 다음 영상으로 스와이프.

### [수동형 호출 시나리오 - 김지혜 씨 모델]
1. 인스타그램 릴스에서 흥미로운 화장품 광고 발견.
2. 구매 의사가 생기자 **[공유하기] → [Ad-Guardian 검증]** 클릭.
3. 팝업창을 통해 "임상 결과 과장 및 AI 생성 영상"이라는 정밀 리포트 확인 후 창을 닫음.

## 3.5 차별점 및 기술적 우위

### 호출 방식 비교

| 구분 | 일반적인 검증 방식 | **Ad-Guardian AI (Active)** |
|------|-------------|-------------|
| **사용자 개입** | 직접 검색, 앱 전환 필요 | **무개입 (자동 감지)** 또는 최소 조작 |
| **대응 시점** | 피해 발생 후 또는 결제 단계 | **광고 시청 단계 (초기 차단)** |
| **핵심 기술** | 텍스트 검색 엔진 | **Accessibility Service + 멀티모달 AI** |

### 경쟁사 대비 경쟁력

- **능동형 방어 체계**: 타 서비스들이 사용자가 앱을 켜서 물어볼 때까지 기다리는 '수동적' 구조라면, 본 서비스는 '찾아가는 검증'을 통해 디지털 숙련도가 낮은 고령층까지 보호할 수 있습니다.
- **Context-Aware 분석**: 현재 사용자가 보는 플랫폼(유튜브/틱톡 등)의 맥락을 유지하면서 오버레이(Overlay) 형태로 정보를 제공하여 쇼핑 경험의 단절을 최소화합니다.

---

### 기획 의도 연결
- 동료 기획자의 **'능동형 호출'** 의견을 반영하여, 페르소나 2(박영수 씨)가 직접 검증 요청을 하지 않더라도 시스템이 보호 기능을 수행하도록 **F1**을 주력 기능으로 상향 조정했습니다.
- 이를 통해 '검색의 귀찮음'과 '인지의 한계'라는 두 타겟의 문제를 동시에 해결하고자 합니다.
